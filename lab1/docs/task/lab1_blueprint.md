2.  []{#_bookmark1 .anchor}**Аналіз скриптів та відтворення даталогічної
    моделі**

Виконаємо відтворення даталогічної моделі за фізичною моделлю за
допомогою скриптів структури.

Скрипти структури містять скрипти для створення таблиць, створення
обмежень цілісності, таких як Primary Key (первинні ключі), Foreign Key
(вторинні ключі), Check, Index.

Даталогічна модель представлена у вигляді схеми на рисунку 2.1.

Даталогічна модель складається з семи таблиць, що містять атрибути та їх
типи а також скорочені позначення обмежень цілісності.

Зв'язки представлені у вигляді стрілок між таблицями з позначками
кардинальності зв'язку.

[Рис. 2.1 -- Даталогічна модель тестової бази даних]{.mark} Дана
даталогічна модель містить:

1.  Обмеження цілісності

    - Первинні ключі (PK)

    - Вторинні ключі (FK)

    - Індекси (І1, І2, І3, І4)

    - Check constraints (CK)

2.  Формалізовані зв'язки (1:M)

3.  Типи даних (NUMBER, DATE, VARCHAR2, CHAR)

4.  Назви об'єктів (таблиці, атрибути)

Деякі таблиці є реалізацією зв'язку M:N між об'єктами.

Створимо власні табличні простори (DEFAULT, TEMPORARY), використовуючи

На рисунку 3.1 -- 3.2 зображені скрипти для створення табличних

> [Рис. 3.1 -- Створення постійного табличного простору]{.mark}

Для створення табличного простору "DEFAULT", що міститиме основні
об'єкти бази даних, використаємо інструкцію CREATE TABLESPACE. Вкажемо
опцію SMALLFILE для табличного простору, що є стандартним для табличного
простору Oracle та може містити 1022 датафайла або темпфайла.

Вкажемо шлях до файлу даних для постійного табличного простору за
допомогою інструкції DATAFILE та розмір табличного простору в 5М
інструкцією SIZE.

Вкажемо параметр AUTOEXTEND ON, що дозволить автоматично збільшити
розмір табличного простору за необхідності на 2М, максимальний розмір
необмежений.

Вказуємо опцію LOGGING, що дозволить реєструвати табличний простір у
redo log файлах.

Задамо значення LOCAL для параметру EXTENT MANAGEMENT, що дозволить
створити табличний простір, що керується локально.

Налаштовуємо автоматичне керування простором сегментів за допомогою
інструкції SEGMENT SPACE MANAGEMENT зі значенням AUTO.

> [Рис. 3.2 -- Створення тимчасового табличного простору]{.mark}

однієї сесії використаємо аналогічну команду вказавши параметр
TEMPORARY.

Вкажемо шлях до тимчасового файлу даних інструкцією "TEMPFILE" та

розмір табличного простору аналогічно до попереднього прикладу.

Вкажемо локальне керування екстентами однакового розміру в 1М за
допомогою інструкції UNIFORM SIZE.
